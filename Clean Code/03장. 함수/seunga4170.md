#많이 쓰는 단항 형식# : 이벤트
이벤트 함수는 입력 인수만 있으며 프로그램은 함수 호출을 이벤트로 해석해 입력 인수로 시스템 상태를 바꾼다. <br> 

함수에 인수 1개를 넘기는 이유 두 가지 <br>
1. 인수에 질문을 던지는 경우 <br>
2. 인수를 뭔가로 변환해 결과를 반환하는 경우 <br> <br>

=> 지금까지의 경우가 아니라면 단항 함수는 가급적 피한다.  <br>

1. 부수 효과를 일으키지 말 것. 
2. 명령과 조회를 분리할 것.
3. 오류 코드보다 예외를 사용할 것.
4. 알고리즘/코드 반복하지 말 것.
